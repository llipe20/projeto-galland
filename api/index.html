Finalidade

O projeto consiste em desenvolver uma mini API para fazer a ponte entre um banco de dados e o front end de uma aplicação pessoal, um sistema online de pedidos/cardápiio. Ou seja, a api servirá apenas para essa aplicação, a menos que queira usar pra testar mesmo.

A aplicação especifica, é a construção de um cardápio online para uma sorveteria, onde pretendo usar esta API para coletar e adicionar os dados de clientes, pedidos e produtos no banco de dados, trazendo mais dinamismo, facilitando o trabalho do desenvolvedor, no caso eu.

Modo de usar

Como havia anteriormente, esta API servirá especificamente para uma única aplicação. Por isso, não contém informações relevantes para uso casual e para outros projetos, como a API de filmes, satélites e etc... Mas, caso queira testar e tals, é bem simples. Basta colocar a url da API - localhost/api/routers/ ... nome da rota, a aplicação possui três rotas, são elas: pedidos, produtos e login- na sua requisição e configurar os método de envio, cabeçalhos e etc...

Pedidos

URL de entrada: copiado
http://localhost/api/routers/pedidos.php

Caso queira os pedidos de um cliente específico, basta passar o id do cliente como um parâmetro no final da URL. Ex: pedidos/2. Assim, a requisição será direcionada apenas um único regristo.

Usuários

Vale ressaltar que não é possivel consultar todos os usuários em uma única requisição, ou seja, a URL deve receber o id do usuário como um parâmetro para que a requisição retorne algo. Ex: login/2.

URL de entrada: copiado
http://localhost/api/routers/login.php/2
Saída: 
{
    "id": Int,
    "nome": String
    "usuario": String,
    "senha": String
}
