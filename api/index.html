Finalidade

O projeto consiste em desenvolver uma mini API para fazer a ponte entre um banco de dados e o front end de uma aplicação especifica. Ou seja, a api servirá apenas para essa aplicação, a menos que queira usar pra testar mesmo.

A aplicação especifica, é a construção de um cardápio online para uma sorveteria, onde pretendo usar esta API para coletar os dados de clientes, pedidos e produtos no banco de dados, trazendo mais dinamismo, facilitando o trabalho do desenvolvedor.

Modo de usar

Como havia anteriormente, esta API servirá especificamente para uma única aplicação. Por isso, não contém informações relevantes para uso casual e para outros projetos, como a API de filmes, satélites e etc... Mas, caso queira testar e tals, é bem simples. Basta colocar a url da API - localhost/api/routers/ ... nome da rota, a aplicação possui três rotas, são elas: pedidos, produtos e login- na sua requisição e configurar os método de envio, cabeçalhos e etc...

Usuários

Vale ressaltar que não é possivel consultar todos os usuários em uma única requisição, ou seja, a URL deve receber o id do usuário como um parâmetro para que a requisição retorne algo. Ex: login/2.

URL de entrada:
http://localhost/api/routers/login.php/2
Saída: 

    {
        "id": "2",
        "nome": "Nara",
        "usuario": "silvahlipe@gmail.com",

Pedidos

Entrada: 

async function newRequest() {
    const data = 
        {
            "id_login":"Int", // id do usuário
            "pedido_valor": "Float", // valor da compra
            "pedido_pagamento": "String", // forma de pagamento
            "pedido_situacao": "String", 
            "pedido_obs": "String", 
            "pedido_delivery": "Boolean", // delivery ou retirada,
            "cliente_id": "Int", // id do cliente
            "cliente_nome": "String",  
            "cliente_email": "String",
            "cliente_fone": "String",
            "cliente_rua": "String",
            "cliente_bairro": "String",
            "cliente_casa": "String",
            "cliente_referencia": "String",
            "produtos": [
                {
                    "produto_id": "Int", // id do produto que está consumindo
                    "quantidade_produto": "Int"
                }
            ],
            "adicionais": [
                {
                    "adicional_id": "Int", // id do adicional que está consumindo
                    "quantidade_adicional": "Int",
                }
            ]
        }

        const url = 'http://localhost/api/routers/pedidos.php'

        req = await fetch(url, {
            method: 'POST',
            headers: 
            { 'Content-Type': 'application/json' },
            body: JSON.stringify(data)
    })
        const res = await req.json()
        console.log(res)
}

Usuários

Neste exemplo de entrada, usei uma função assíncrona para fazer a requisição POST. Os dados de entrada são estes da variável data. Vale lembrar que a rota é a mesma, o que muda é a penas o method, que deve ser configurado para POST.

Entrada: 

async function newUser() {
    const data = 
        {
            "nome" : "String",
            "email" : "String",
            "senha" : "String"
        }

        const url = 'http://localhost/api/routers/login.php'

        req = await fetch(url, {
            method: 'POST',
            headers: 
            { 'Content-Type': 'application/json' },
            body: JSON.stringify(data)
    })
        const res = await req.json()
        console.log(res)
}
Saída: 

    {message: 'Usuario criado com sucesso.'} or
